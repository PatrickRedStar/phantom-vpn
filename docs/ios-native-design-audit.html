GhostStream iOS audit

Нативный iOS дизайн вместо Android-порта

Android версия задает полезную информационную архитектуру: Stream, Tail, Settings, live telemetry, фильтры логов и профильные действия. Admin в GhostStream не глобальный раздел, а право конкретного VPN профиля. Но iOS не должен выглядеть как Compose внутри SwiftUI. Этот макет сохраняет бренд GhostStream и данные VPN, но строит интерфейс на привычных iOS паттернах: NavigationStack, TabView, cards как grouped surfaces, native sheets, toolbar actions, крупные touch targets и понятные состояния.

Что сейчас выглядит не по-iOS

Проблемы найдены по SwiftUI коду iOS и сверке с Android Compose.

Навигация
  • Нижний bar повторяет Android glyph language: ◉, ▤, ⚙. Для iOS лучше SF Symbols + TabView semantics.
  • Глобальный Admin ломает модель продукта: у обычного покупателя доступа его не должно быть вообще. Нативнее: profile detail -> server control только для admin profile.
  • Swipe between tabs есть, но iOS-пользователь ожидает стабильный TabView без случайных горизонтальных конфликтов со ScrollView.
Типографика
  • Огромный terminal headline хорошо работает на Android, но на iPhone съедает первый экран.
  • ALL CAPS labels везде создают cockpit feel; iOS лучше читает sentence case + secondary labels.
  • Моноширинный текст нужен для логов и чисел, но не для всего Settings и metadata.
Controls
  • Rectangular GhostToggle и GhostChip выглядят как custom control kit, не как iOS.
  • Dialogs вручную рисуются поверх ZStack; для iOS лучше sheet, confirmationDialog, alert с кастомным content только где реально надо.
  • Connect FAB снизу хороший, но его визуал должен быть iOS primary button, а не Android block button.
Информационная плотность
  • Stream экран одновременно показывает headline, timer, chart, mux, KV и CTA. На iPhone это надо разбить на hero card + раскрываемые sections.
  • Settings уже функционально насыщен, но визуально похож на набор кастомных rows вместо iOS grouped settings.
  • Logs хороши как tail, но на iOS нужен search/filter affordance и более крупная readable row cadence.

Решение для реализации

Это не “натянуть Android”, а сохранить parity по функциям.

  1. Системная навигация без глобального Admin Использовать SwiftUI TabView только для Stream, Logs, Settings. Admin не вкладка и не пункт для всех: он появляется только в карточке конкретного профиля, если у этого профиля есть admin/cert entitlement.
  2. Stream как iOS status card Главный экран: large title, connection card, компактные metrics, chart, route/profile details. Не делать огромный Android headline на половину экрана.
  3. Settings как grouped form Grouped cards/sections с native toggles, disclosure rows, confirmationDialog для delete, sheet для import/DNS/split-routing.
  4. Logs как читаемый tail Newest-first остается, но filters должны быть iOS capsule chips, ShareLink, Clear confirmation, optional search.
  5. Brand без костылей Оставить warm-black, lime accent, graph/mux visuals. Убрать Android text glyphs, custom rectangular switches и hand-built modal chrome.

Философия профилей

GhostStream продает доступ к конкретным VPN endpoints. Admin — это свойство endpoint/profile, а не свойство приложения.

Правило видимости Admin Если активный или сохраненный профиль обычный, пользователь видит только подключение, логи и настройки своего доступа. Если у профиля есть admin certificate/capability, в карточке именно этого профиля появляется action “Server control”. Никакой глобальной вкладки Admin, никаких пустых admin screens для покупателя доступа.
  1. Обычный профиль Card actions: Connect, Edit, Share to device, Delete. Admin affordance отсутствует полностью, не disabled.
  2. Admin профиль Card actions: Connect, Server control, Edit, Export client config, Delete. Badge “Admin” показывает capability только на этой карточке.
  3. Admin screen lifecycle Открывается через NavigationLink из admin profile card. Back возвращает к профилю. Header показывает имя сервера и mTLS identity этого профиля.

SwiftUI mapping

Компоненты, которые можно воплотить точно и без “рисования Android руками”.

NavigationStack + TabView Root navigation for Stream, Logs and Settings only; Admin is pushed from an admin-capable profile.
.toolbar + .navigationTitle Large titles, profile menu, diagnostics/share actions without custom header duplication.
List / ScrollView + Section Settings, profile detail and profile-scoped Admin grouped rows with native spacing and disclosure indicators.
sheet(detents:) + confirmationDialog Import, DNS, split routing, delete profile/client and copy/share flows.
ShareLink + Photos/Camera sheet Logs export, debug report, QR import use platform affordances.
Canvas or Swift Charts Traffic chart can remain branded but should sit inside an iOS card hierarchy.
