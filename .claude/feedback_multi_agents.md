Пользователь хочет, чтобы при крупных изменениях (>5 файлов или >3 независимых зон) я запускал параллельных субагентов через Agent tool в одном сообщении.

**Why:** v0.19.4 релиз по плану stateless-puzzling-wall был выполнен через 4 параллельных агента (Dev-Server, general-purpose core, Dev-Android, general-purpose docs) — в ~3× быстрее, чем если бы я редактировал все файлы сам. Пользователь явно одобрил подход после завершения ("делай и используй мультиагентов").

**How to apply:**
- Разделять задачу по файловым зонам так, чтобы агенты не пересекались (non-overlapping edits) — иначе получим merge conflicts в рабочем дереве.
- Типичная декомпозиция для этого репо:
  - **Dev-Server** — `crates/server/`
  - **Dev-Android** — `android/`, `crates/client-android/`
  - **Dev-Linux** — `crates/client-linux/`
  - **general-purpose (core)** — `crates/core/`, `crates/client-common/`
  - **general-purpose (docs)** — `*.md`, `config/`, `scripts/`
- Агенты запускаются одним tool-call'ом с несколькими `Agent` блоками — это параллель, а не серия.
- После выполнения — самому прогнать `cargo check -p <crate>` и `cargo test` для валидации (агенты могут пропустить cross-crate compile errors).
- Для точечных правок в одном файле (<50 строк, 1 файл) — субагент не нужен, делать инлайн.

**Sign of task fit for multi-agent:** ≥3 crate'ов/модулей, низкая взаимозависимость правок, можно сформулировать задачу каждому агенту ≤200 слов.
